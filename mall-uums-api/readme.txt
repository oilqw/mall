产品名称：统一认证系统
用途特点：统一认证系统是门户网站建设的基础，主要实现用户管理、身份认证、分级权限管理和单点登录等功能，以解决门户建设过程中用户定义模糊、用户身份组织零乱、交叉权限管理和应用系统出口多样性等棘手问题。
具备以下功能：用户信息规范明了，统一存储，用户ID全局惟一；UUMS向各应用系统提供用户属性列表；应用系统对用户基本信息的增加、修改、删除和查询等请求由UUMS处理；应用系统保留用户管理功能；UUMS具有完善的日志功能。 
特点： 
支持web service技术框架 
方便使用，能尽可能地利用现有系统的身份认证模块以及现有的用户设置和权限设置，尽量保护现有的资源。 
具有良好的扩展性和可集成性，不仅能支持现有的应用系统及其现有的用户系统，当有新的企业应用被部署或开发的时候，这个统一身份整整服务可以作为它的身份认证模块的形式工作。 
具备灵活和方便的使用模式，使用者可以通过多种方式自由地使用该统一身份认证服务。性价比：用户资料的统一存储和管理。 
用户身份集中验证。 
采用基于身份认证的单点登录，可显著提高信息化的易用性，降低用户的使用难度，提高用户的使用效率。价格优惠：按市场同期价格的5%给予优惠产品图片：
应用案例：统一身份认证系统将机构、用户统一存储，对应用系统统一授权、规范应用系统的用户认证方式。 
吉林银行统一身份认证系统解决方案 
（1）将内部柜员和外部客户的身份认证从各个应用系统分离出来，形成一个统一的身份认证平台，身份认证对各个应用系统是透明的； 
（2）采用模块化的设计，认证平台除了支持静态密码和指纹认证外，还能方便的增加双因素、PKI等多种认证机制，实现了指纹、静态密码、动态口令、智能令牌、密码钥匙等多种认证方式 ； 
（3）统一共用原则：支持多业务系统共用，以认证平台作为银行身份认证门户。 
（4）实现所有新建应用系统的单点登录。
